- 対象取引: 2025-12-16 株式会社イチネンパーキング OP今宮駅前 駐車料金 600円
- 調査範囲: 現金レシート1件の読取結果、弥生会計23向け仕訳データ、エラーログ、

## 1. 結論

今回の仕訳は、最終的に弥生会計23へ正常にインポートされた。仕訳日記帳では、
税区分が「課対仕入10%」、請求書区分が「適格」、仕入税額控除が「100%」であることを
ユーザーが確認しており、当該仕訳を再インポートする必要はない。

一方で、今回の対応は1件の出力ファイルを手作業で修正したものであり、
SHINKOKUの生成ロジックを恒久的に修正したものではない。次の2点が未解決である。

1. 仕訳の識別フラグと取引タイプを整合させる弥生出力処理が、現行ツリーに実装されていない。
2. 適格請求書発行事業者の登録番号、取引日時点の有効性、請求書区分、
   仕入税額控除割合を判定して弥生税区分へ明示する処理がない。

この状態では、次回以降も弥生会計側の補完処理に依存する可能性がある。
科目設定の初期値が取引実態と異なる場合には、誤った請求書区分または控除割合で
取り込まれるリスクがある。

## 3. 初回インポート失敗の原因

初回出力では、識別フラグを `2111`、取引タイプを `0` としていた。
弥生会計のエラーログには、次の内容が記録された。

```text
1行目,識別フラグと取引タイプが矛盾しています。
```

弥生インポート形式では、`2111` は「1行の伝票データ」を表す一方、
取引タイプ `0` は「仕訳データ」を表す。今回のデータは伝票ではない通常仕訳であるため、
識別フラグを `2000`、取引タイプを `0` とする組み合わせが適切である。

初回失敗の直接原因は拡張子ではなく、この組み合わせの矛盾であった。
ただし、弥生会計23での運用に合わせ、修正版は `.txt` として作成した。

この修正版を使用した結果、弥生会計23のインポートは正常終了した。

## 5. 正常終了後に表示された警告の分析

正常終了時には、次の警告が表示された。

```text
弥生インポート形式と一致しない請求書区分、仕入税額控除があったため、
科目設定の初期値にあわせて設定しました。
```

修正版の借方税区分は `課対仕入込10%` であり、請求書区分を示す末尾の
`適格`、または区分記載請求書と控除割合を示す `区分80%` などを指定していない。

弥生会計の仕様では、請求書区分と仕入税額控除の記述を省略してもインポートできる。
その場合には、取引日付や科目設定を基に弥生会計が値を補完する。
今回の警告は、この補完処理が行われたことを通知するものであり、
インポート失敗を示すエラーではない。

インポート後にユーザーが仕訳日記帳を確認した結果は次のとおりである。

| 項目 | 補完後の値 |
|---|---|
| 税区分 | 課対仕入10% |
| 請求書区分 | 適格 |
| 仕入税額控除 | 100% |

今回の仕訳結果は取引内容と整合している。ただし、この結果は弥生会計が登録番号を検証した
ものではなく、「旅費交通費」などの科目設定の初期値を採用した結果である。

## 6. 現行実装の不足

### 6.1 レシート読取結果に登録番号がない

`skills/reading-receipt/SKILL.md` の `RECEIPT_DATA` は、現在、次の項目を返す。

- `date`
- `vendor`
- `total_amount`
- `tax_included`
- `items`

適格請求書発行事業者の登録番号、税率別金額、消費税額、請求書区分、
仕入税額控除割合を返す項目がない。このため、後続処理が適格請求書かどうかを
根拠付きで判定できない。

### 6.2 現行ツリーに弥生出力の恒久処理がない

現行の `src/`、`skills/`、`tests/` を検索した範囲では、弥生会計向けの
出力処理またはテストは確認できなかった。今回の `.txt` は調査中に手作業で生成したものである。

したがって、「今回の1件が正常に取り込めたこと」と「今後も正しいファイルを自動生成できること」は
区別する必要がある。

## 7. 推奨する判定フロー

弥生会計側の補完処理に依存しないためには、レシート読取から弥生出力までの間に
次の判定を追加する必要がある。

1. レシートから `T` + 13桁の登録番号を読み取る。
2. 国税庁の適格請求書発行事業者公表サイトまたはWeb-APIで番号を照合する。
3. 登録年月日、取消年月日、失効年月日を使用し、取引日時点で有効だったかを確認する。
4. 発行者名と公表情報の名称を照合する。
5. 税率、税込・税抜、取引日、納税者の課税方式を判定する。
6. 少額特例、帳簿のみ保存の特例、免税事業者からの仕入れに係る経過措置を判定する。
7. 判定結果を弥生税区分の末尾へ明示する。
8. 判定不能またはOCR結果が不確実な場合には、自動確定せず「確認要」とする。

今回のレシートについて、登録番号が取引日時点で有効であることを確認できた場合の
借方税区分は、次の形式になる。

```text
課対仕入込10%適格
```

適格番号がない、または取引日時点で無効であり、2025年取引について経過措置の80%控除を
適用する場合の例は次のとおりである。ただし、少額特例などの適用可否を先に確認する必要がある。

```text
課対仕入込10%区分80%
```

## 8. 推奨するデータ項目

レシート読取結果または中間モデルへ、少なくとも次の項目を追加することが望ましい。

| 項目案 | 内容 |
|---|---|
| `invoice_registration_number` | `T` + 13桁の登録番号 |
| `invoice_issuer_name` | レシート上の発行者名 |
| `invoice_number_verified` | 国税庁での照合結果 |
| `invoice_valid_on_transaction_date` | 取引日時点の有効性 |
| `invoice_verified_at` | 照合日時 |
| `tax_rate` | 10、8など |
| `tax_included` | 税込・税抜 |
| `invoice_category` | 適格、区分記載、対象外、確認要 |
| `input_tax_credit_rate` | 100、80、70、50、30、0など |
| `classification_reason` | 判定根拠 |
| `source_file` | 元レシート画像のパスまたは識別子 |

登録番号が印字されていることだけでは、適格請求書としての確定条件にならない。
取引日時点での有効性、適格簡易請求書を含む必要記載事項、納税者側の課税方式と特例を
合わせて判定する必要がある。

## 9. 推奨する検証

恒久対応を実装する場合には、少なくとも次のテストが必要である。

1. 通常仕訳が `2000` と取引タイプ `0` の組み合わせで出力される。
2. 1行が25列になり、CP932・CRLFの `.txt` として出力される。
3. 10%税込の適格請求書が `課対仕入込10%適格` になる。
4. 軽減税率8%の適格請求書が対応する軽減税率表記になる。
5. 取引日時点で無効な登録番号を「適格」として出力しない。
6. 2025年の区分記載請求書について、条件を満たす場合に80%控除を設定する。
7. 少額特例などが適用される取引を、単純に区分80%へ分類しない。
8. OCRで登録番号を確定できない場合には、自動出力を停止して確認を求める。
9. 弥生会計へのインポート時に、識別フラグのエラーと請求書区分の補完警告が発生しない。

## 10. 制度・製品バージョン上の注意

本調査の対象は2025年12月16日の取引である。免税事業者等からの仕入れに係る控除割合は、
2026年10月以降に段階的に変更されるため、割合を固定値として実装してはならない。
現行リポジトリには80%から70%、50%、30%、0%へ移行する制度資料が含まれている。

また、弥生会計23はインボイス制度への対応版であっても、後年の制度改正へ継続対応するとは
限らない。2026年10月以降の取引を出力する場合には、使用中の弥生会計のバージョンが
該当する税区分と控除割合を受け付けるかを別途確認する必要がある。
